  │ 指标  │                                                  含义                                                  │
  ├───────┼────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ PPL   │ Perplexity，语言模型困惑度。越低说明模型对序列的预测越准确                                             │
  │ Len   │ 生成序列的平均长度                                                                                     │
  │ >20%  │ 生成序列中长度超过 20 的比例。真实数据范围是 [5, 20]，所以越低越好                                     │
  │ 5-20% │ 生成序列中长度在 5-20 范围内的比例。越接近 100% 越好                                                   │
  │ AA_KL │ 生成序列的 20 种氨基酸频率分布 vs 训练数据的 KL 散度。越低说明 AA 组成越接近真实蛋白质                 │
  │ Uniq% │ 2500 条生成序列中唯一序列的比例。越高多样性越好                                                        │
  │ Top1% │ 出现频率最高的那条序列占总数的比例。高说明模式坍缩                                                     │
  │ EDist │ 随机抽 500 对生成序列算 Levenshtein 编辑距离的均值。越高说明序列间差异越大                             │
  │ Novel │ 生成的唯一序列中不在训练集里的比例。越高说明模型不是在背诵训练数据                                     │
  │ RT_Fl │ Round-trip Flattened Spearman：输入相图 → 生成序列 → 预测相图，把所有样本所有 16 个位置展平算 Spearman │
  │ RT_Mn │ Round-trip Mean Spearman：同样往返，但比较的是每个样本 16 维预测值的均值之间的排序相关                 │
  └───────┴────────────────────────────────────────────────────────────────────────────────────────────────────────┘
  1. Uniform：长度从 [5, 20] 均匀随机采，每个位置从 20 种 AA 等概率随机选
  2. FreqMatch：长度从真实训练数据的长度分布中采样，每个位置按真实 AA 频率采样（L 13%、S 10.4% 这种）
  3. ShuffleReal：直接从训练集随机抽 2500 条真实序列，把每条序列内的 AA 顺序打乱

     ============================================================================================================================================
     关键发现
     ============================================================================================================================================
  1. Set encoder (per-position velocity) 在 Seq→Phase Spearman 上 **全面胜出** (12/12)，
     平均提升 +0.028，最大 +0.042 (missing3)。消除 cross-attn 信息瓶颈是关键。

  2. Cross-RT 独立验证 Set encoder 胜率 9/12，最佳 set_flow5_m11 Cross_RT_Mn=0.496，
     说明 Set encoder 生成的序列在独立 Seq2Phase 模型下也能更好地恢复原始相图。

  3. set_flow0 组 Cross-RT 高达 0.40-0.50 (Self-RT≈0)，与 legacy flow0 类似，
     说明 LM 方向学到了 phase→seq 条件映射，set encoder 改善了 phase 条件编码。

  4. flow_weight 对 LM 质量的影响：
     - flow0 (纯 LM): Cross-RT 最高 (0.495)，但 flow 方向不可用
     - flow1/flow5: 兼顾两个方向
     - flow32: flow 过强压制 LM，Cross-RT 降低

  5. 生成序列质量（AA_KL, 多样性, 新颖性）两种架构基本持平，Set encoder 未引入退化。
